    目的

    针对研发平台和产品线的 MSF 实践团队，提供 UE 代言群在各个历程的实施指导，明确关键活动及其要求，为 UE 角色在MSF团队中顺利开展工作提供帮助和指导。

    阅读者及阅读建议

    • 产线MSF团队的 UE

    • 软件部UE

    范围

    本文档主要涉及到MSF各个历程的UE关键活动、关键输出及评审要求。

    历程指导

    展望历程

    展望历程需要完成的主要活动是组成核心团队和撰写远景/范围文件，而远景/范围文件需要包括用户信息和验收标准，则需要 UE 参与以下工作：

    1. 协助产品管理收集、分析、定义用户需求，验证用户需求和高层需求一致。
    2. 协助架构设计产品的用户交互。
    3. 协助产品管理收集、归纳用户信息。
    4. 验收标准中的用户验收标准需要UE来制定。

    关键输出物

    • 远景/范围文件之用户验收标准。

    关键活动

    • 制定用户验收标准
    1. 专注于易用性和功能。
    2. 明确基于用户代表的验收范围及其标准。
    • 参与评审用户信息
    1. 确认用户类别。（比如通过分析用户知识、技能、能力、目标、动机、关注点等对用户进行分类）
    2. 明确用户代表及归纳整理。
    • 文档化工作及评审
    • 展望历程输出的文档化工作（编写及检查）。

    计划历程

    计划历程专注于构建的内容、构建的方式、构建每个部件的时间、 交付解决方案需要的支持环境。该阶段UE需要关注用户需求的细分，包括本地化需求、可访问性需求、易用性等需求，以及关注用户使用场景，用户培训及手册的时间表等。

    1. 协助定义、细化用户需求及优先级。
    2. 协同编写/完善需求规格说明书（主要涉及到易用性相关的用户使用场景及用例）。
    3. 制定培训计划。

    关键输出物

    • 培训或制定用户手册计划

    关键活动

    • 协同分解及细化需求
    1. 从展望阶段的高层需求，分解为基于用户使用场景的具体需求。
    2. 定义需求的优先级，如：必须有的、应该有的、可有可无的。
    • 制定培训计划
    1. 明确受众、培训方式、培训时间、培训时机、培训资源。

    构建历程

    构建历程根据计划历程的交付成果来构建解决方案的各个方面。该阶段UE需要关注：

    1. 协同开发代言群沟通，提供明确的用户交互设计。
    2. 协同 UI 设计师完成原型及图形界面元素的设计。
    3. 初步构建用户手册、培训资料和易用性验收测试场景。
    4. 参与功能规格说明书的评审。

    关键输出物

    • 无

    关键活动

    • 无

    稳定历程

    稳定历程主要目标为了 提升解决方案的质量，满足利益相关人的需求和期望。该阶段UE需要基于用户的角度验收解决方案的易用性。

    1. 用户验收测试及提供验收测试报告。
    2. 制定并完善客户和用户手册。
    3. 培训试运行用户。

    关键输出物

    • 验收报告
    • 用户手册
    • 培训资料

    关键活动

    • 执行验收测试
    • 按照远景/范围文件确定的验收标准，执行验收测试。
    • 制定用户手册

    • 产品的用户手册包括用户手册、帮助文件、技术支持文档、故障诊断手册、操作手册等。

    • 培训用户

    • 培训用户代表或其他试用的用户。

    部署历程

    部署历程主要目标将解决方案集成到指定的业务环境中，交付解决方案给运营团队。该阶段UE需要提供培训和技术支持。

    1. 协同/支持售后用户培训。
    2. 同市场/销售部沟通，确认用户设计、沟通用户问题反馈及制定改进措施。
    3. 协同接收产品升级反馈、问题反馈及跟踪、协同制定改进措施，提高用户满意度。
    4. 协同MSF开发、测试复现问题、跟踪问题得到解决。

    关键输出物

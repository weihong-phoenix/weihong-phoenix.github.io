    目的

    控制产品文档的编写、审核、存放、印刷过程，使文档的管理规范化。

    范围

    本规范适用于以下中英文产品文档：

    • 厂商手册
    • 用户手册
    • 快速上手指南
    • 功能手册
    • 编程手册
    • 维宏百问

    命名规则

    对于产品文档面向的对象及专业技能的要求不同，产品的用户文档分为不同的类型，以下为各种类型文档的定义。

    【规则】：

    • 厂商手册，面向机床厂商的调机技术人员，内容主要关注软件支持的工控机型号、硬件连接及软硬系统调机。分为以下厂商手册：

      • 运动控制卡系列

        运动控制卡型号+系统名称+“数控系统厂商手册”+Rn，例如：PM95A玻璃切割数控系统厂商手册-R1

      • 工业计算机系列

        工业计算机型号+系统名称+“数控系统厂商手册”+Rn，例如：NC60A义齿机数控系统厂商手册-R1

      • 一体机系列：一体机型号+系统名称+“集成数控系统厂商手册”+ Rn，例如：NK280B外圆磨床集成数控系统厂商手册- R1

    • 用户手册,面向机床加工操作人员，内容主要关注数控软件的功能操作步骤、驱动器参数介绍及其调试。一般包括软件用户手册和驱动器软件及参数用户手册。

      • 控制系统类软件：软件名称+系统名称+ “控制系统用户手册”，例如： NcStudio Phoenix石材加工中心控制系统

      • 驱动器：维智WSDA系列直线伺服驱动器用户手册（脉冲型）-R1；iMotion操作手册-R3

    • 快速上手指南

      • 厂商手册快速上手指南 根据厂商手册的命名规格命名后，将“厂商手册”改成“快速上手指南”。

        • 户手册快速上手指南 根据用户手册的命名规格命名后，将“用户手册”改成“快速上手指南”。例如：NK300BX集成数控系统快速上手指南- R1

    • 功能手册，待更新 功能名称+“使用说明”+“-Rn”

      例如：侧挂式圆盘刀库调试使用说明-R1

      问题：功能手册和用户手册的差别在哪里？为什么不把功能手册放到用户手册的某个功能里面。

    • 编程手册，待更新 软件名称+“编程手册”+ Rn

      例如：NcStudio 编程手册-R1

    • 维宏百问： “维宏数控系统百问”+“厂商\用户手册”+ Rn 例如：维宏数控系统百问厂商\用户手册-R1

    文档内容

    基于不同类型的文档，其结构、内容和表达风格不尽相同。

    表达风格

    参考产品研发文档编写指南。

    文档架构

    • 所有文档必须包括目的、范围、术语定义。

    • 所有产品文档必须包括产品简介。包括产品定位及特点。可以通过示意图或列表方式介绍。

    • 所有文档涉及到操作步骤的必须通过图示和文字综合表达的方式陈述。

    • 所有文档必须有常见问题，必须包括现象、原因、处理方法。

    不同类型文档及内容

    文档内容必须准确无误(主要针对中文文档)，表达无歧义，包括内容的正确，相应图片的正确和更新。 英文翻译须遵守语法规则，内容准确无误。 对于需要印刷的文档最后一章需加上“用户安装软件许可声明”。

    厂商手册

    厂商手册包含但不限于以下内容：

    • 接线：介绍端子板和接线方法等。
    • 操作：按照功能分模块介绍软件。每个模块大概从概念及原理、操作过程、参数三个方面介绍，在操作过程中要设置哪些参数、遇到问题时应该怎样调整参数、参数之间有什么联系。
    • 调试与维护：介绍操作系统恢复、Ncstudio安装与打包、列表说明每种报警的产生原因、解决办法、常见故障处理。
    • 驱动器：主要介绍与机床连接相关的参数、驱动器与端子板接线图。
    • 参数一览：列表说明所有参数，其中详细说明见“操作”章节介绍，“操作”中没有涉及到的参数在此做简要说明。 除了以上必须要包含的内容，不同板卡的厂商手册可以针对板卡特有的功能进行介绍。

    用户手册

    用户手册中包含但不限于以下内容：

    • 概述：说明软件特性
    • 系统安装：说明系统的基本配置和安装方法、硬件更新驱动步骤
    • Ncstudio基本概念
    • 软件操作界面
    • 软件菜单系统
    • 软件参数介绍
    • 带有NcEditor的软件介绍NcEditor的功能和特点

    快速上手指南

    快速上手指南必须包含但不限于以下内容：

    • 系统简介：说明系统功能、组成、软件布局等。
    • 快速开始：说明系统快速运行的流程，及每个流程的具体操作。
    • 常见问题：列出运行过程常见问题的原因及解决方法。

    功能手册

    功能手册的内容包含但不限于以下内容：

    • 引言：说明文档的描述对象、内容及作用。
    • 功能简介：描述功能对象是什么、有什么特点、在什么情况下能用到。
    • 操作指导：功能手册的主体内容，基于使用场景，生成任务型的小节。
    • 常见问题处理：不是必写项，对在以上设置、操作过程中遇见的常见问题进行说明。

    各项内容的详细说明及写作风格参见《功能手册模板》。

    编程手册

    编程手册必须包含但不限于以下内容：

    • 数控编程概述
    • 零件程序的结构
    • 编程指令体系
    • 自定义扩充M指令
    • PLT支持、DXF支持

    维宏百问

    维宏百问手册必须包含但不限于以下内容：

    • 标题必须写明可以解决的问题是什么。
    • 每个标题下的正文，解答问题产生的原因、影响及如何解决问题。

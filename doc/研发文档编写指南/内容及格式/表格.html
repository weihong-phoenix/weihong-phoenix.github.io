    表格

    使用原则

    1. 如果可以用其它标签，则尽量不使用表格。

    2. 表格要严格遵守简洁、一致性，同类型表格必须使用同样的表格样式。

      参数/信号/报警等类型的信息，必须使用表格。

    基本要求

    1. 表格标题和内容都必须简洁，避免长篇大论，特别是标题。
    2. 不要出现空白表格，无需内容的，统一填写 “-” 。
    3. 出现多个单元格内容一样的，将内容复制，不要写 “同上” 。
    4. 同一列信息，表述方式一致，结尾的标点也要一致。
    Note

    表格可以通过直接编辑标签或通过其它工具插入（如：typora），无论何种方式创建的表格，为了确保表格的一致性，要求通过 Learn Markdown 插件的功能进行格式化。即通过插件里面的选项 Consolidate selected table 去除创建表格时额外多余的空格。

    具体使用参考章节：仓库及发布工具->VS Code->支持格式化表格。

    应用

    参数/信号型

    参数/信号类型的表格可以按照以下样式进行描述。

    Note

    所有参数及信号去掉 ”名称“，所有参数的概述性信息放到 ”描述“ 中。

    Note

    表格里面不要有 “备注” 信息，如果需要，请直接在 “描述” 里面。

    报警型

    报警类型的表格可以按照以下样式进行描述。

    • 编号，参数编号规则暂未定。
    • 类型，包括警告、错误、提示等类型。
    • 内容，报警提示信息。
    • 说明，根本原因及解决方法。
    编号 类型 内容 说明
    10000 警告 加工过程中不可以设定最短路径 原因分析：在加工过程中修改了最短路径。
    解决办法：请在非加工状态下修改该参数。
    10001 错误 加工过程中不可以设定最短路径，XXXXX 原因分析：在加工过程中修改了最短路径。
    解决办法：请在非加工状态下修改该参数。

    字体

    默认字体

    .md 格式文档的默认正文字体。

    粗体

    粗体适用于描述界面元素的标题、正文中需要强调的内容，包括 菜单选项 、对话框名称 、输入框的 Label 、参数名称等，以及这些内容需要在正文中引用的情况。

    正文中若有其它非此类词语或句子需要重点强调，期望引起读者一定程度重视，也可以使用粗体。

    如果比粗体更加希望引起重视，则可以用 安全标识 章节的 注释 标签。

    正例 反例
    点击 开始 按钮， 启动自动加工。 点击开始按钮， 启动自动加工。

    另外，粗体前后是否需要空格，有争议，不做特别要求，一个文档内保持一致即可，从文档美观我个人比较推荐前后有空格，特别是 UI 界面的元素。

    斜体

    斜体适用于描述以下内容：

    • 带有定义或解释的术语简介。
    • 文件名称、文件夹名称、路径。
    • 用户输入。
    正例 反例
    将 D:\Weihong\NcStudio\ActiveConfig 复制到优盘中。 将D:\Weihong\NcStudio\ActiveConfig复制到优盘中。
    通过 CNC.json 文件构建 HandwheelDevice 时可通过 axescount 指定控制轴的数量。 通过CNC.json文件构建 HandwheelDevice时可通过 axescount指定控制轴的数量。
    输入 排样_XXX 作为文件名称，点击 保存 按钮。 输入排样_XXX作为文件名称，点击 保存 按钮。

    内联样式 ``

    正文的描述信息中，需要对某一类型的文本进行区分，并非想重点强调的，则可以用 XXX 进行弱强调。 比较典型的案例是寻址名称、函数名称，且前后各需一个空格隔开。

    但本身对该寻址的定义描述则不需要。比如信号、参数章节对该寻址则不需要 `` 。

    正例 反例
    手轮设备可以通过 #HandwheelDevice.Direction 来控制手轮是正转还是反转。 手轮设备可以通过 #HandwheelDevice.Direction 来控制手轮是正转还是反转。

    正例:

    手轮设备信号，脉冲计数

    项 指
    寻址 #HandwheelDevice.PulseCount
    标题 脉冲计数
    描述 手轮设备摇动所产生的脉冲计数。
    类型 int

    反例：

    手轮设备信号，脉冲计数

    项 指
    寻址 #HandwheelDevice.PulseCount
    标题 脉冲计数
    描述 手轮设备摇动所产生的脉冲计数。
    类型 int

    此处非正文信息的寻址引用，因此不需要另外加 `` 进行标识。

    英文大小写

    句首字母大写

    • 告警或提示符号的首字母必须大写。如：Error，Warnning，Tips等。

    • 完整句子首个单词首字母必须大写。如：状态栏弹出提示、操作提示或说明、参数解释或描述等。
    • 列表中的描述性信息首字母必须大写。

    单词首字母大写

    • 单个词语或短语中的每个单词首字母必须大写。
    • 单选或复选的选项为一个完整的句子时，仅句首字母大写。
    • 标识为常显状态或模式的文字首字母必须大写。
    • 菜单及菜单选项的首字母必须大写。

    坐标和通用缩写全部大写

    • 坐标轴都需要大写：X，Y，Z，……

    • 通用缩写都需要大写：IO（Input/Output） ，MFR（Multi-frequency Receiver），HW（Hand Wheel），USB（Universal Serial Bus），……

    • 文件格式描述都需要大写：PLT，DXF，ENG，……

    小写的情况

    • 控件上的短语中，介词（如of，by，to，in，on，at，for，as等）或连词（如and，or，with等）一般需小写，如“Set as Default”中“as”需小写。

    • 通用短语 “Save As…（存储为…）” s中的 “As” 首字母需要大写。

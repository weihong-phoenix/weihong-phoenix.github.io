    标题

    合适的标题可以引导读者，让读者快速领会内容的核心。

    标题层级

    标题层级严禁 超过四级。

    标题必须从 一级标题 开始，逐级递增使用，禁止跳级。

    一级

    可以和目录名称一致。文档中若没有其它层级分类，仅有一级标题，可直接正文描述。

    二级

    正文的标题，二级标题根据内容可有多个，比如描述功能规格的，包括概述、场景、参数等，同为二级标题。

    三级

    三级标题根据文档内容可有多个，比如场景描述可以分为正常场景、异常场景，同为三级标题。

    四级

    一般文档三级标题即可有逻辑的完成正文的描述，不宜采用四级标题。

    Note

    除一级标题应仅有一个，其它级别标题至少有两个。即一级标题后，不得只有一个二级标题，此种情况应直接为正文，其它级别同理。

    最小层级

    项目列表是最小编号单位，因此项目列表下禁止嵌套任何级别的标题。

    Caution

    文档名称和标题都禁止编号，若需要编号，可以在配置文件 toc.yml 中编辑。

    描述

    长度限制

    中文标题长度不得超过 24 个中文字符（包括英文半角字符和空格）。

    说明：一个英文半角字符或空格，当作半个中文字符对待。

    不重复

    下级标题不应重复上级标题已包含内容。

    正例 反例
    # 回机械原点
    ## 原理介绍
    ## 参数说明
    ## 常见问题及调整    		 # 回机械原点
    ## 回机械原点原理介绍
    ## 回机械原点参数说明
    ## 回机械原点常见问题及调整

    结构统一

    相同级别、相同类型的标题结构保持一致。

    • 描述类采用名词性结构
    正例 反例
    NcStudio 简介 介绍NcStudio
    • 过程类采用动宾结构
    正例 反例
    安装运动控制卡 运动控制卡的安装

    • 标题应无需上下文则能准确识别其含义，概括反映章节的中心内容，应简洁无歧义。

      举例说明：

      1. 产品名称不作为标题的一部分。
      2. 回参考点、加工向导、设定坐标系。
      3. 主题词+动词，比如：编码器锁存、仿真加工、圆盘对刀、台面编辑。

    标题描述并无严格模板，可遵循以下原则：

    • 标题应概括反映主题思想。
    • 标题应简洁扼要、涵义明确。
    • 同级别标题应使用相同结构。

    面向用户

    标题设计应从读者角度出发，直接回答一个用户关心的问题。

    正例 反例
    安装 NcStudio 如何安装NcStudio？
    调整轴方向 如何调整轴方向？

    符号约束

    标题中关于符号的约束：

    • 标题中直接若使用缩略语，不包含带括号的缩写词全称。一般不建议在标题中解释缩略语。

      正例 反例
      IP on Top of IpoE IP on Top of IpoE (IP over Ethernet)

    • 标题禁止以句号、问号等表示句子意思结束的标点结尾。（不适用FAQ、案例标题）

    • 标题中不应带上下标。

    • 禁止在标题中用“&”表示并列；中文要使用“和”或“与”，英文要使用“and”或“or”表并列。

    • 禁止在标题中用“/”表示“与”和“或”的关系；应明确使用“与”、“或”、“and”、“or”表示两者之间的关系。

      正例 反例
      Introduction to PM62A and PM63A Introducing the PM63A/62A

    • 若标题中出现界面元素、命令字、商标、产品名称、专有名词、术语等，可不受符号规则的约束。

    • 禁止在标题中用连字符或英文斜杠表补充说明，应使用冒号或小括号表示。

      正例 反例
      Modify Operator (MOD OP) Modify Operator --MOD OP
      1.3.6.1.4.1.2011.5.2.2.2.0.3: Failing to Set the QOS Template of the Users 1.3.6.1.4.1.2011.5.2.2.2.0.3-- Failing to Set the QOS Template of the Users

    • 标题禁止直接编号，若需要编号，可以在配置文件toc.yml中编辑name标签，而内容顺序同样通过toc进行配置即可。

    • 若完成一个过程可以使用多种方式，可以在过程类标题中使用圆括号简要说明不同的方法。

    例如：

    Installing the System (Command Line)

    Installing the System (GUI)

    Installing the System (Web Browser)

    其它

    • 故障的定位、处理和解决相关对标题必须包含故障现象。
    • 用户场景类标题（如安装、配置等）采用任务风格，基于用户真实的任务场景。
    • 标题要避免孤立编号（即同级标题只有一个），正文不要有孤立的三级标题和四级标题。
    • 标题与标题之间应该有引导介绍性的句子，即概要描述。

    一级标题和二级标题之间应有引言内容，二级标题和三级标题之间应有正文内容。

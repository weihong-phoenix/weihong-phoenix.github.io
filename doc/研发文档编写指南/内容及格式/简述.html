    简述

    简述，即对标题的简短描述，英译Short Description。标题之后的简单概要描述。

    长度

    • 简述必须为 1～3 句话，不超过 60 个字。
    正例 反例
    访问控制列表
    接入控制列表可提供基础的数据流过滤功能。配置了 ACL 的网元都可在IP报文通过时选择是否过滤此报文。ACL 可控制特定数据流传入或传出某个网络。    		 访问控制列表
    这个主体是主要描述了访问控制列表的概念。

    正例也超出了 60 字，一般不推荐，但为了描述清楚，如果语言已经足够精炼，字数可以放宽。

    反例太过于简单模糊不清，无法通过简述了解改部分核心功能。

    类型

    概念类

    概念类主题的简短描述，介绍概念的同时要回答“是什么？”和“为什么需要关注？”

    正例 反例
    访问控制列表
    接入控制列表可以提供基础的数据流过滤功能。
    所有配置了 ACL 的网元都可以在IP报文通过时选择是否过滤此报文。
    ACL 可以控制特定的数据流传入或传出某个网络。    		 访问控制列表
    这个主体是主要描述了访问控制列表的概念。

    参考类

    参考主题的简短描述，简单说明“参考项有哪些？” “分别是什么或者有什么作用？”。

    正例 反例
    COUNT command
    The COUNT command displays the current number of rows in the table.
    The rows are counted by the SQL SELECT COUNT(*) function.    		 COUNT command
    DB2 Universal Database provides a COUNT command.

    任务类

    任务类的简短描述，要求解释任务的目的和对用户带来的好处，包括帮助用户理解何时任务是适用的，为什么任务是必须的信息。但不采用“你可以使用 XYZ 完成 A ” 这种简单但无用的描述。

    正例 反例
    恢复网管数据
    在网管数据损坏或要将已有数据导入新安装的网管时，可以使用备份的 MO 数据恢复网管的数据。该操作将覆盖网管当前的所有数据。    		 恢复网管数据
    你可以使用 Restore 命令恢复网管数据。

    其它

    • 避免简单拷贝正文的第一段话。

    • 避免使用简短描述作为标题的引导语。避免使用“本章”、“本节”作为简短描述的主语或者缺省主语。

      Short Description 避免简单写成“本节描述了系统安装的主要内容”。也不能写成“描述了系统安装的主要内容”（缺省主语为本节）。

    • 避免简单重复标题。

      反例：

      相关性分析

      完成相关性分析。（简短性描述）
